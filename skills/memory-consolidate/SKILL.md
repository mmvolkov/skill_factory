# Консолидация памяти проекта

Периодический «разбор архива». Неверный или дублирующийся паттерн хуже отсутствующего: он даёт ложное узнавание.

## Чек-лист (по порядку)

1. **Дубли.** Карточки об одном факте слить в одну; ссылки `[[...]]` перенаправить.
2. **Устаревшее.** Несущие факты каждой карточки сверить с текущим состоянием репозитория; опровергнутое — исправить или удалить.
3. **Выводимое.** Содержимое, которое теперь есть в коде/доках/README, из памяти убрать (оставить только невыводимое «Why»).
4. **Ссылки и индекс.** Починить `[[ссылки]]` на удалённые карточки; в `MEMORY.md` — ровно одна строка на живую карточку, хуки соответствуют содержимому.
5. **Карта.** Разросшиеся разделы карты проекта выгрузить в карточки; из «Открытых нитей» удалить закрытые, даты — абсолютные.

## Выход

Краткий отчёт: сколько карточек слито/удалено/обновлено, что осталось подозрительным.
