# /note — заметки Obsidian-хранилища

Управляет заметками по соглашению хранилища: **канонная копия живёт в vault, дубликат — в `notes/` проекта**, обе копии держим синхронными. Заметки пишутся **по-русски**.

Аргументы: `$ARGUMENTS`

## Константы путей

- **VAULT (канон):** `D:\projects\obsidian\vault`
  Заметка: `D:\projects\obsidian\vault\<project>\<note>.md`
- **Проект (дубль):** `D:\projects\<project>`
  Заметка: `D:\projects\<project>\notes\<note>.md`

`<note>` по умолчанию равен `<project>` (как `diarization/diarization.md`). Если эти пути когда-нибудь изменятся — поправь их здесь.

## Дата

Везде используй **сегодняшнюю абсолютную дату** в формате `YYYY-MM-DD`, бери её из контекста сессии (`currentDate`). Никогда не пиши «сегодня»/«вчера» в сохраняемый текст.

## Диспетчеризация по аргументам

Разбери `$ARGUMENTS` (первый токен — команда). Если пусто или команда не распознана — покажи **статус хранилища**.

### `new <project> ["<title>"]` — создать заметку

1. `<note>` = `<project>` (если пользователь не задал иное имя файла).
2. `<title>` = переданный заголовок, иначе осмысленный из названия проекта.
3. Проверь, существует ли `VAULT\<project>\<note>.md`. Если да — не перезаписывай, сообщи и предложи `status`/редактирование.
4. Создай папку `VAULT\<project>\` при необходимости и запиши канон по **шаблону** (ниже), подставив реальные пути и сегодняшнюю дату в первую строку `## Статус`.
5. Дубликат: если папка проекта `D:\projects\<project>\` существует — создай `D:\projects\<project>\notes\` при необходимости и запиши туда **идентичную** копию. Если папки проекта нет — создай только канон и предупреди, что дубликат пропущен.
6. Сообщи, какие файлы созданы (оба пути).

### `status <project> [<note>] "<text>"` — добавить запись в журнал

1. Найди канон `VAULT\<project>\<note>.md` (по умолчанию `<note>`=`<project>`). Если нет — предложи `new`.
2. В секцию `## Статус` добавь строку `- YYYY-MM-DD — <text>` (новые записи идут в конец списка, историю не переписывай).
3. Примени **то же** изменение к дубликату `D:\projects\<project>\notes\<note>.md`, если он есть.
4. Сообщи, что обновлено.

### `task <project> [<note>] "<text>"` — добавить задачу

1. Как `status`, но добавляй `- [ ] <text>` в секцию `## Задачи` обеих копий.

### `sync [<project>]` — сверить канон и дубликаты

Канон (vault) — источник истины.

1. Определи область: для одного `<project>` или для всех папок проектов в `VAULT` (кроме `.obsidian`, `.claude`).
2. Для каждой заметки сопоставь канон и дубликат:
   - **Только в каноне** → создай дубликат (с `notes/` при необходимости), если папка проекта существует.
   - **Различаются** → покажи краткий diff и, после подтверждения, перезапиши дубликат содержимым канона (перезапись необратима — сначала покажи разницу).
   - **Только в дубликате** → не удаляй молча; сообщи как осиротевший и спроси: повысить до канона (скопировать в vault) или оставить.
   - **Совпадают** → ничего не делай.
3. Выведи сводку: создано / обновлено / пропущено / конфликты.

### нет аргументов / неизвестная команда — статус хранилища

1. Перечисли проекты (папки в `VAULT`, кроме `.obsidian`, `.claude`) и заметки в них.
2. Для каждой заметки отметь наличие дубликата и состояние синхронизации (совпадает / различается / дубликата нет).
3. Подскажи доступные команды.

## Шаблон новой заметки

```markdown
# <Title>

> Эта заметка дублируется в двух местах:
> - vault (канон): `D:\projects\obsidian\vault\<project>\<note>.md`
> - проект (дубль): `D:\projects\<project>\notes\<note>.md`

## Статус
- YYYY-MM-DD — Заметка заведена.

## Описание
<Назначение проекта. Если неизвестно — отметь это явно, не выдумывай.>

## Соглашения по заметкам
- Все статусы / знания / задачи / заметки / описания ведём в Markdown.
- Канонная копия — в vault: `D:\projects\obsidian\vault\<project>\`.
- Дубликат — в проекте: `D:\projects\<project>\notes\`.

## Задачи
- [ ] <Первая задача.>
```

## Замечания по реализации

- **Всегда читай файл перед правкой** (Edit/Write) и переноси изменение в обе копии в одном проходе, чтобы они не разъезжались.
- Блок-цитату с двумя путями в начале заметки сохраняй и держи пути актуальными.
- Можно использовать идиомы Obsidian: `[[вики-ссылки]]`, callouts, YAML-frontmatter (плагин properties включён).
- Удаление/перезапись — только после показа пользователю, что именно меняется.
