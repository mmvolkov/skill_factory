# Промоция паттерна в скилл

Память проектов — попроектная; носитель знаний между проектами — скиллы. Паттерн, повторившийся в трёх проектах, — кандидат в скилл (аналог слияния чанков в темплейт).

## Процедура

1. **Собрать свидетельства.** Найти вхождения паттерна: карточки памяти и карты разных проектов, повторяющиеся решения/конфиги. Порог — три независимых вхождения (пользователь может понизить явным запросом).
2. **Отделить инвариант от специфики.** В скилл идёт только переносимое; проектные детали остаются в памяти проектов.
3. **Написать скилл как темплейт, а не справочник:** триггеры (когда узнавать ситуацию) → типовой план → типовые риски/грабли. Frontmatter: `name` + `description` с триггерами.
4. **Разместить в двух местах:** канон — `D:\projects\skill_factory\skills\<name>\SKILL.md`; рабочая копия — `~/.claude/skills/<name>/SKILL.md`. Обновить README фабрики.
5. **Зафиксировать веху** в памяти skill_factory: какой паттерн, из каких проектов, дата.

## Гигиена

- Не промоутить паттерн по одному яркому случаю — ложный темплейт хуже отсутствующего.
- При изменении скилла синхронизировать обе копии.
