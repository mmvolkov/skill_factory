# skill_factory

Фабрика собственных скиллов-навыков для Claude Code: здесь разрабатываются, обкатываются и хранятся канонические версии скиллов, которые делают работу агента в любых проектах более качественной — особенно после перерыва в работе или перезагрузки, когда контекст приходится восстанавливать с нуля.

## Идея

Проект вырос из разбора статьи [«Шахматная память»](https://habr.com/ru/companies/lanit/articles/1043140/) (Хабр, ЛАНИТ): экспертная память — это не «суперпамять», а библиотека распознаваемых структур — чанков и темплейтов. Перенос на агента:

- **скилл = темплейт** — «узнал ситуацию → знаешь типовой план и типовые риски»;
- **память проекта = внешняя долговременная память**, переживающая перезагрузку; работает, только если хранит структуру («как устроено и почему»), а не хронику сессий;
- **качественное решение = четыре фазы де Гроота** — ориентировка, кандидаты, анализ, проверка.

## Скилл note — подробно

`note` — второй, человекочитаемый контур долговременной памяти. Если память агента (`~/.claude/projects/<project>/memory/`) читает только Claude Code, то заметки `note` живут в Obsidian-хранилище и предназначены прежде всего для человека: статусы, задачи и знания по проектам в одном vault, с графом кросс-связей между проектами.

**Соглашение о двух копиях.** У каждой заметки два экземпляра, которые держатся синхронными:

- **канон** — `D:\projects\obsidian\vault\<project>\<note>.md` (vault — источник истины);
- **дубль** — `D:\projects\<project>\notes\<note>.md` (заметка видна прямо в репозитории проекта).

По умолчанию `<note>` совпадает с именем проекта. Все записи ведутся по-русски, даты — только абсолютные (`YYYY-MM-DD`), «сегодня/вчера» в сохраняемый текст не пишутся.

**Команды:**

| Команда | Действие |
|---|---|
| `/note new <project> ["<title>"]` | Создать заметку по шаблону (канон + дубль); существующую не перезаписывает |
| `/note status <project> "<text>"` | Добавить датированную строку в журнал `## Статус` обеих копий |
| `/note task <project> "<text>"` | Добавить чекбокс `- [ ]` в секцию `## Задачи` обеих копий |
| `/note sync [<project>]` | Сверить канон и дубли: создать недостающие, показать diff расхождений, найти осиротевшие |
| `/note` (без аргументов) | Статус хранилища: проекты, заметки, состояние синхронизации |

**Структура заметки:** шапка с обоими путями → `## Статус` (журнал с датами, история не переписывается) → `## Описание` → `## Соглашения по заметкам` → `## Задачи` (чекбоксы). Поддерживаются идиомы Obsidian: `[[вики-ссылки]]` (ими и строятся кросс-связи между проектами в графе vault), callouts, YAML-frontmatter.

**Правила безопасности:** файл читается перед правкой, обе копии меняются в одном проходе; при `sync` перезапись дубля — только после показа diff; осиротевшие дубли не удаляются молча, а предлагаются к повышению до канона.

**Место в семействе:** `memory-discipline` + `project-map` — память *для агента* (структурированное узнавание после перезагрузки), `note` — память *для человека* (обзор всех проектов в Obsidian). Кросс-проектные знания агента переносятся скиллами через [skill-promote](skills/skill-promote/SKILL.md), кросс-проектные знания человека — вики-ссылками vault.

## Устройство

- `skills/<name>/SKILL.md` — каноническая версия скилла (этот репозиторий);
- `~/.claude/skills/<name>/SKILL.md` — рабочая копия, которую подхватывает Claude Code во всех проектах.

Изменения синхронизируются между репозиторием и `~/.claude/skills` в обе стороны вручную.

## Соглашения

- Один скилл = папка `skills/<name>/` с единственным `SKILL.md`.
- Скилл пишется как темплейт, а не справочник: триггеры → типовой план → типовые риски.
- Новые скиллы появляются через [skill-promote](skills/skill-promote/SKILL.md): паттерн должен повториться примерно в трёх проектах, прежде чем стать скиллом.
