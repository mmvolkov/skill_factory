# skill_factory

Фабрика собственных скиллов-навыков для Claude Code: здесь разрабатываются, обкатываются и хранятся канонические версии скиллов, которые делают работу агента в любых проектах более качественной — особенно после перерыва в работе или перезагрузки, когда контекст приходится восстанавливать с нуля.

## Идея

Проект вырос из разбора статьи [«Шахматная память»](https://habr.com/ru/companies/lanit/articles/1043140/) (Хабр, ЛАНИТ): экспертная память — это не «суперпамять», а библиотека распознаваемых структур — чанков и темплейтов. Перенос на агента:

- **скилл = темплейт** — «узнал ситуацию → знаешь типовой план и типовые риски»;
- **память проекта = внешняя долговременная память**, переживающая перезагрузку; работает, только если хранит структуру («как устроено и почему»), а не хронику сессий;
- **качественное решение = четыре фазы де Гроота** — ориентировка, кандидаты, анализ, проверка.

## Скиллы

| Скилл | Назначение |
|---|---|
| [memory-discipline](skills/memory-discipline/SKILL.md) | Дисциплина проектной памяти: ритуал ориентировки, чанкинг, запись на вехах, гигиена |
| [project-map](skills/project-map/SKILL.md) | Карта проекта — один файл фиксированной структуры, точка входа ориентировки |
| [decide](skills/decide/SKILL.md) | Четырёхфазный протокол рискованных решений: кандидаты и проверка перед действием |
| [skill-promote](skills/skill-promote/SKILL.md) | «Правило трёх»: паттерн из трёх проектов промоутится в новый скилл |
| [memory-consolidate](skills/memory-consolidate/SKILL.md) | Консолидация памяти проекта: дубли, устаревшее, ссылки, индекс |

## Устройство

- `skills/<name>/SKILL.md` — каноническая версия скилла (этот репозиторий);
- `~/.claude/skills/<name>/SKILL.md` — рабочая копия, которую подхватывает Claude Code во всех проектах.

Изменения синхронизируются между репозиторием и `~/.claude/skills` в обе стороны вручную.

## Соглашения

- Один скилл = папка `skills/<name>/` с единственным `SKILL.md`.
- Скилл пишется как темплейт, а не справочник: триггеры → типовой план → типовые риски.
- Новые скиллы появляются через [skill-promote](skills/skill-promote/SKILL.md): паттерн должен повториться примерно в трёх проектах, прежде чем стать скиллом.
